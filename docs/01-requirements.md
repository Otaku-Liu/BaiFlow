# BaiFlow 需求说明

## MVP 目标
第一版 MVP 要完成一个可运行的个人文件流转闭环：用户可以登录 Web 管理台，管理服务器本地文件，创建下载任务，并在 Android App 中完成登录、文件查看、上传和下载。

BaiFlow 现在采用“权限模型提前设计、功能分阶段落地”的策略。多用户、分享 URL、隐私文件夹会进入整体设计；实现时先做角色和认证基础，再逐步落地分享与隐私访问。

## 用户与角色
### 管理员 ADMIN
- 管理用户、角色和系统设置。
- 管理全部 Storage Root、文件、下载任务、分享链接和审计日志。
- 可重置普通用户密码、禁用用户、撤销分享链接。

### 普通用户 USER
- 登录 Web 和 Android。
- 访问管理员授权给自己的文件、文件夹或 Storage Root 范围。
- 创建自己的上传、下载和分享任务，具体能力受权限配置限制。
- 访问隐私文件夹时仍需输入隐私密码。

### 访客 GUEST
- 不作为完整登录用户参与管理台。
- 主要通过分享 URL 访问指定文件或文件夹。
- 只能执行分享链接允许的只读访问或下载。
- 如分享目标受隐私保护，必须额外输入访问密码或提取码。

## MVP 功能
### 认证与权限
- 管理员初始化。
- 用户名密码登录。
- JWT 访问令牌。
- 密码加密存储。
- 用户角色：ADMIN、USER、GUEST。
- 未登录用户不能访问受保护 API。
- 普通用户只能访问授权范围内资源。

### 文件中心
- 配置一个服务器本地 Storage Root。
- 浏览目录和文件。
- 上传文件。
- 下载文件。
- 新建文件夹。
- 重命名文件或文件夹。
- 移动文件或文件夹。
- 删除文件或文件夹。
- 文件元数据写入 MySQL。
- 文件夹可标记为隐私文件夹。
- 隐私文件夹需要额外访问密码，密码只保存 hash。

### 分享访问
- 支持对指定文件生成分享 URL。
- 支持对指定文件夹生成分享 URL。
- 分享链接可设置过期时间、最大访问次数、最大下载次数。
- 分享链接可设置提取码。
- 分享访问不暴露服务器真实路径。
- 分享链接可被管理员或创建者撤销。

### 下载中心
- 通过 aria2 RPC 创建 URL、磁力、BT 下载任务。
- 查看任务状态。
- 暂停、恢复、删除任务。
- 下载完成后写入文件索引。
- 记录失败原因。

### 传输与通知
- 统一展示上传、下载任务。
- 状态包括等待、运行、暂停、失败、完成。
- Web 内通知中心展示任务完成和失败。

### Android Java App
- 登录服务器。
- 查看授权范围内的文件列表。
- 上传手机文件。
- 下载服务器文件。
- 查看任务状态。
- 长时间传输使用前台通知。

## 非目标
- 第一版不做团队协作空间。
- 第一版不做云盘级在线预览。
- 第一版不直接实现 SMB、WebDAV、NFS 客户端。
- 第一版不自研 BT 或磁力下载协议。
- 第一版不做复杂自动化流程编排。
- 第一版不做细粒度 ACL 到每个操作按钮；先实现角色和资源范围控制。

## 后续阶段
- NAS 挂载目录接入。
- WebDAV 访问。
- 自动归类和简单规则引擎。
- 更细粒度用户权限和家庭成员空间。
- 分享访问统计和分享审计报表。
- 外网访问安全加固。
