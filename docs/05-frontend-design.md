# BaiFlow Web 前端设计

## 技术栈
- Vue 3
- Vite
- Vue Router
- Pinia
- Axios
- Element Plus 或 Naive UI

## 页面结构
### 登录页
- 用户名输入
- 密码输入
- 登录按钮
- 登录失败提示

### 主布局
- 左侧导航
- 顶部状态栏
- 主内容区
- 通知入口
- 用户退出入口

### 文件中心
- Storage Root 切换
- 面包屑路径
- 文件列表
- 上传按钮
- 新建文件夹
- 重命名、移动、删除操作
- 下载按钮

### 下载中心
- 新建下载任务表单
- 下载任务列表
- 状态、进度、速度、错误信息
- 暂停、恢复、删除操作

### 传输中心
- 上传、下载、设备流转任务统一列表
- 状态筛选
- 失败任务错误展示

### 设备中心
- 已注册设备列表
- 设备类型
- 最近在线时间
- 状态

### 通知中心
- 通知列表
- 未读标记
- 标记已读

### 系统设置
- Storage Root 配置
- aria2 RPC 配置
- Nginx 外部访问地址提示
- 上传大小限制提示

## 前端目录建议
```text
baiflow-web/src/
  api/
  assets/
  components/
  layouts/
  router/
  stores/
  views/
  utils/
```

## 状态管理
- `authStore`：token、用户信息、登录状态
- `fileStore`：当前 Storage Root、当前目录、文件列表
- `transferStore`：传输任务和实时进度
- `notificationStore`：通知和未读数量

## UI 原则
- 管理台以信息密度和可扫描性为主。
- 文件列表优先使用表格。
- 危险操作必须二次确认。
- 长任务必须显示状态和错误原因。
- 空状态要说明下一步操作，例如上传文件或创建下载任务。

## API 调用原则
- Axios 统一注入 token。
- 401 统一跳转登录页。
- 请求失败统一显示错误消息。
- 文件上传显示进度。
- 文件下载使用浏览器下载能力，不把大文件读入内存。
