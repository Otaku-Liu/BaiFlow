# BaiFlow 项目概览

## 项目名称
- 英文名：BaiFlow
- 中文名：小白流转
- App 名称：BaiFlow
- 服务端：baiflow-server
- Web 前端：baiflow-web
- Android 端：baiflow-android

## 项目定位
BaiFlow 是一个运行在个人服务器上的下载与文件协同中心。它以服务器本地磁盘作为第一版文件根据地，围绕文件流转、下载任务、设备协同、传输记录和通知构建个人数据中枢。

第一版不追求替代完整网盘，也不追求复杂团队协作。目标是让一个人稳定、安全、可控地管理自己的下载、上传、设备文件流转和服务器文件归档。

## 目标用户
- 拥有个人服务器、NAS、家庭小主机或云服务器的个人用户。
- 希望用 Web 管理台和 Android App 管理文件、下载任务和传输任务的用户。
- 熟悉基础服务器部署，愿意使用 Docker Compose、Nginx、MySQL 的用户。

## 核心场景
1. 在 Web 管理台创建下载任务，服务器下载完成后自动进入文件中心。
2. 在 Android App 上传手机文件到服务器文件根据地。
3. 在 Web 管理台浏览、移动、重命名、删除服务器文件。
4. 查看上传、下载、失败、完成等传输状态。
5. 后续把 NAS 挂载目录接入为额外存储根目录。
6. 后续通过分享链接临时分发文件。

## 产品原则
- 稳定优先：每个阶段都要能独立运行和验收。
- 安全优先：默认私有访问，不暴露数据库和服务器真实路径。
- 小步交付：先文件中心和下载中心，再 Android，再 NAS 和分享。
- 不重复造轮子：BT、磁力和多协议下载交给 aria2。
- 文件本体落磁盘，数据库只存元数据。
