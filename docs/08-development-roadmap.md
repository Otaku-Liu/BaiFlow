# BaiFlow 开发路线图

## 推进原则
- 一次只做一个阶段。
- 每个阶段都必须可运行、可验收。
- 不提前实现下一阶段功能。
- 每个阶段完成后更新文档。
- 权限模型、分享、隐私文件夹先设计清楚，再分阶段实现。

## Phase 0：文档与边界
交付：`docs/` 文档体系。
验收：能说明项目目标、技术方案、需求边界、API、数据库、部署、安全和阶段计划。

## Phase 1：项目骨架
交付：后端、Web、部署目录。
验收：后端 `/api/health` 可访问，Web 首页可打开，MySQL 可连接。

## Phase 2：认证、用户与基础权限
交付：登录、JWT、用户管理、角色模型、接口鉴权。
验收：未登录不能访问受保护 API；管理员可创建普通用户；普通用户只能访问授权资源；访客角色为分享访问预留。

## Phase 3：文件根据地 MVP
交付：本地 Storage Root、文件浏览、上传、下载、移动、删除、基础授权校验。
验收：Web 端完成基础文件管理，路径安全测试通过，普通用户不能越权访问。

## Phase 3.5：隐私文件夹
交付：文件夹隐私密码、访问验证、短期访问会话。
验收：隐私文件夹即使对已登录用户也要求额外密码；密码错误不能访问；修改密码后旧会话失效。

## Phase 4：下载中心 MVP
交付：aria2 接入、下载任务管理。
验收：Web 创建下载任务，完成后文件进入文件中心，任务归属创建者。

## Phase 5：传输中心与通知
交付：统一任务中心、进度展示、通知中心。
验收：上传下载进度可见，失败原因可见，通知可读。

## Phase 6：Android Java MVP
交付：Android 登录、文件列表、上传、下载、任务状态。
验收：真机或模拟器能完成移动端文件流转闭环，普通用户只能看到授权文件。

## Phase 7：NAS 接入
交付：NAS 挂载目录作为 Storage Root。
验收：可浏览 NAS 文件，NAS 离线时系统不崩溃。

## Phase 8：分享 URL 与访问控制
交付：文件/文件夹分享链接、过期时间、访问次数、下载次数、提取码、分享访问日志。
验收：分享受限可控；分享文件夹可浏览；隐私文件夹分享仍需隐私密码或提取码；分享可撤销。

## Phase 9：安全与部署加固
交付：HTTPS、审计、生产部署文档、登录失败限制、分享访问限流。
验收：部署安全基线满足要求，敏感配置走环境变量。

## 测试策略
- 后端每阶段补充单元测试和接口测试。
- Web 每阶段做手动验收，关键组件后续补测试。
- Android 每阶段使用真机或模拟器手动验收。
- 部署每阶段至少本地 Docker Compose 启动一次。

## 关键安全测试
- 未登录访问文件接口返回 401。
- 普通用户访问未授权文件返回 403。
- 访客不能访问非分享资源。
- 分享链接过期后不可访问。
- 分享下载次数达到上限后不可下载。
- 提取码错误不能访问分享内容。
- 隐私文件夹密码错误不能访问。
- 删除文件不会误删 Storage Root 外的路径。

## 暂缓事项
- 团队空间。
- 复杂自动化规则。
- 分片上传和断点续传。
- WebDAV 服务端。
- 直接 SMB/NFS 客户端。
